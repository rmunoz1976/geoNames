## Objetivo
Procesar el archivo TSV adjunto e insertar los datos en la tabla MySQL `t076t_states`.

## Estructura de la tabla destino: `t076t_states`

 Campo  Tipo  Obligatorio  Valor / Regla 
-----------------------------------------
 `id`  int AUTO_INCREMENT  NO  Lo genera la BD 
 `geo_id`  int  SÍ  Columna 1 (geonameid) 
 `id_country`  int  SÍ  Usar valor de `id_country_origen` (ver datos al final) 
 `code`  varchar(64)  SÍ  `{code_pais}-{admin1_code}` (Columna 11) 
 `code_dane`  varchar(16)  NO  NULL 
 `code_dian`  varchar(16)  NO  NULL 
 `code_iso_3166_2_alpha2`  varchar(16)  SÍ  Mismo valor que `code` 
 `code_iso_3166_2_alpha3`  varchar(16)  SÍ  Mismo valor que `code` 
 `phone_code`  varchar(16)  SÍ  Usar según `code_pais` (ver datos al final) 
 `code_postal`  varchar(16)  SÍ  'NULO' 
 `id_division_type`  int  SÍ  Usar valor de `id_division_type_origen` (ver datos al final) 
 `name_spanish`  varchar(128)  SÍ  Columna 4 (alternatenames): buscar variante en español → si no, usar Columna 2 (name) 
 `name_english`  varchar(128)  SÍ  Columna 3 (asciiname) 
 `name_iso`  varchar(128)  SÍ  Columna 2 (name) 
 `name_original`  varchar(128)  SÍ  Columna 2 (name) 
 `name_transcribed`  varchar(128)  SÍ  Columna 3 (asciiname) 
 `status`  int  NO  1 (activo) 
 `created_at`  datetime(6)  SÍ  NOW(6) 
 `updated_at`  datetime(6)  SÍ  NOW(6) 
 `created_by`  int  SÍ  1 
 `updated_by`  int  SÍ  1 

## Estructura del archivo TSV

 Columna  Nombre  Uso 
----------------------
 1  geonameid  `geo_id` 
 2  name  `name_iso`, `name_original`, respaldo para `name_spanish` 
 3  asciiname  `name_english`, `name_transcribed` 
 4  alternatenames  Fuente principal para `name_spanish` 
 5-10  latitude, longitude, feature_class, feature_code, country_code, cc2  No se usan 
 11  admin1_code  Para construir `code` 
 12-19  admin2_code, admin3_code, admin4_code, population, elevation, dem, timezone, modification_date  No se usan 

## Reglas específicas
- Usa utf8mb4; escapa comillas simples DENTRO de valores de texto
duplicándolas (''), NO con barra invertida (\').
Ejemplos de nombres que requieren escape:
  M'sila      → 'M''sila'
  Aïn N'Sour  → 'Aïn N''Sour'
  Oum el Bouaghi → sin apóstrofe, no requiere escape
Aplica a TODOS los campos varchar: name_spanish, name_english,
name_iso, name_original, name_transcribed, code, etc.
Antes de generar el SQL, revisar todos los valores en busca de
apóstrofes o comillas simples embebidas y duplicarlas.
- NULL sin comillas para valores nulos
- name_spanish en minúsculas con tilde si aplica
- name_english siempre en inglés sin caracteres especiales
- name_iso: nombre oficial exacto según ISO 3166-2
- No inventes geo_id; si no encuentras el dato real de geonames.org, indícalo
- Si el país tiene subdivisiones de distinto tipo (ej: regiones + provincias),
agrupa por tipo y genera un INSERT de division_type por cada uno


### Para `name_spanish`
- Buscar en `alternatenames` (columna 4) palabras con:
  - Tildes (á, é, í, ó, ú, ñ)
  - Formas españolas como "San", "Santa", "Ciudad", "Provincia"
- Si se encuentra una variante en español, usarla
- Si no, usar `name` (columna 2)
- Si `name` no es legible, usar `asciiname` (columna 3)

### 2. Para campos de auditoría
- `created_at` y `updated_at`: Siempre `NOW(6)`
- `created_by` y `updated_by`: Siempre `1`

## Formato de salida

Generar un script SQL con el siguiente formato, con contenido listo para DESCARGAR:

1. **Primera línea**: `-- {nombre_archivo_origen}_INS` (donde nombre_archivo_origen es el nombre del archivo TSV sin extensión)
2. **Línea siguiente**: `-- DETALLE DE name_spanish POR REGISTRO:`
3. **Por cada registro**: una línea con `-- - {name_original}: {explicación de cómo se obtuvo name_spanish}`
4. **Sentencia INSERT** con todos los registros
5. **Total de registros a insertar
